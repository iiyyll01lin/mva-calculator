## 1. 執行摘要

### 1.1 目的
本文件定義了一套自動化 Pipeline 的設計規格，用於從 SOP 文件與作業影片生成 MiniMOST 工時分析表。系統結合電腦視覺 (Vision Stream) 與自然語言處理 (Logic Stream)，產出標準化的時間量測數據。

### 1.2 範圍
- **輸入**: SOP 文件 (PDF/Image)、作業影片 (MP4)、配置資料
- **輸出**: MiniMOST CSV 分析表、差異報告、驗證結果
- **目標精確度**: TMU 計算準確度 90-95%，配合人機協作驗證

### 1.3 核心功能
| 功能       | 說明                    |
|------------|-------------------------|
| 雙軌分析   | Vision + Logic 並行處理 |
| 多模態融合 | 結合影片數據與 SOP 規則 |
| 自動驗證   | 基於規則的品質檢查      |
| 人機協作   | IE 工程師審核介面       |

## 3. 資料結構規格

### 3.1 表格一：Video Ground Truth (v3.2)

**用途**: 記錄 AI 視覺分析結果 (實測數據 As-Is)

| 欄位            | 型別    | 必填 | 說明                  | 範例                                                                        |
|-----------------|---------|------|-----------------------|-----------------------------------------------------------------------------|
| `action_id`     | String  | 是   | 動作唯一識別碼        | `V-01`, `V-05.01`                                                           |
| `time_sec`      | Object  | 是   | 開始/結束時間 (秒)    | `{"start": 0.0, "end": 2.0}`                                                |
| `key_frames`    | Object  | 是   | 關鍵幀標記 (S/C/R/E)  | `{"start": 0, "contact": 15, "release": null, "end": 60}`                   |
| `cycle_group`   | String  | 否   | 循環群組 ID           | `CG-01`                                                                     |
| `sop_id`        | String  | 是   | 對應 SOP Logic 的參照 | `S-02b`                                                                     |
| `hand`          | Enum    | 是   | 手部使用              | `RH`, `LH`, `BH`                                                            |
| `object_state`  | Object  | 是   | 物件狀態變化          | `{"before": {"DIMM": "tray"}, "after": {"DIMM": "cleaned"}}`                |
| `most_sequence` | Object  | 是   | MOST 參數與來源       | `{"A": {"value": 1, "source": "video"}, "G": {"value": 1, "source": "db"}}` |
| `tmu`           | Integer | 否   | 時間量測單位          | `50`, `220`                                                                 |
| `simo_overlap`  | String  | 否   | SIMO 重疊時間         | `1.8s (LH Hold)`                                                            |
| `confidence`    | Float   | 是   | AI 信心度 (0-1)       | `0.92`                                                                      |
| `review_status` | Enum    | 是   | 審核狀態              | `Pending`, `Approved`, `Review`                                             |
| `anomaly`       | Object  | 否   | 異常標記              | `{"type": "Method Change", "severity": "Info"}`                             |

**JSON Schema**:
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["action_id", "time_sec", "key_frames", "sop_id", "hand", "object_state", "most_sequence", "confidence", "review_status"],
  "properties": {
    "action_id": {"type": "string", "pattern": "^V-[0-9]{2}(\\.[0-9]{2})?$"},
    "time_sec": {
      "type": "object",
      "properties": {
        "start": {"type": "number"},
        "end": {"type": "number"}
      }
    },
    "confidence": {"type": "number", "minimum": 0, "maximum": 1},
    "review_status": {"enum": ["Pending", "Approved", "Review", "Rejected"]}
  }
}
```

## 4. 驗證規則

### 4.1 P0 驗證規則 (必要)

| 規則 ID    | 名稱          | 條件                                                                            | 錯誤處理                 |
|------------|---------------|---------------------------------------------------------------------------------|--------------------------|
| **VAL-01** | 循環完整性    | 每個 `CG-XX` 必須有 3 個動作 (拿取 → 對準 → 下壓)                               | 標記 `Data Incomplete`   |
| **VAL-04** | 頻率一致性    | `COUNT(DISTINCT cycle_group)` 必須等於 `Config Lookup.total_dimm` 或產生 Diff   | 自動產生 `Config Diff`   |
| **VAL-07** | SOP ID 唯一性 | `SELECT COUNT(*) FROM SOP_Logic GROUP BY sop_id HAVING COUNT(*) > 1` 必須回傳 0 | 拒絕匯入                 |
| **VAL-09** | 元件覆蓋      | Video GT 中所有 `object` 必須存在於 Component DB                                | 標記 `Component Missing` |

### 4.2 P1 驗證規則 (建議)

| 規則 ID    | 名稱       | 條件                                                     | 錯誤處理               |
|------------|------------|----------------------------------------------------------|------------------------|
| **VAL-02** | TMU 合理性 | 單一動作 TMU 必須在 [10, 500] 範圍內                     | 標記待審核             |
| **VAL-05** | 時間連續性 | `action[n].end_sec ≈ action[n+1].start_sec` (±0.5s 容許) | 警告                   |
| **VAL-06** | 信心度檢查 | 若 `confidence < 0.7` 且 `review_status != 'Approved'`   | 自動設定 `Review` 狀態 |
| **VAL-08** | 證據存在性 | `evidence` URL 必須可存取                                | 警告                   |
| **VAL-10** | 時間順序   | `V-[n].end_sec < V-[n+1].start_sec` (±0.1s 容許)         | 警告                   |

## 5. 處理流程

### 5.1 階段一：資料攝取

| 步驟 | 輸入            | 處理                            | 輸出            | 時間         |
|------|-----------------|---------------------------------|-----------------|--------------|
| 1.1  | 影片 (MP4)      | 降頻至 5-10 fps，提取關鍵幀      | 幀序列          | ~2 分鐘/影片 |
| 1.2  | SOP (PDF/Image) | OCR + 排版分析                  | 結構化 SOP JSON | ~30 秒/頁    |
| 1.3  | 配置            | 依 Model+SKU 查詢 Config Lookup | Freq、DIMM 數量  | <1 秒        |

**前置驗證閘門**:
- 影片解析度 ≥ 1080p
- SOP 必填欄位完整
- 版本號對應正確

### 5.2 階段二：雙軌分析

#### 視覺軌道
| 元件      | 技術                         | 輸入     | 輸出            |
|-----------|------------------------------|----------|-----------------|
| 手部偵測  | YOLOv8-Pose (自訓練手套模型) | 影片幀   | 手部邊界框      |
| 物件偵測  | YOLOv8                       | 影片幀   | 物件邊界框      |
| 動作辨識  | Temporal CNN                 | 幀序列   | 動作類型 + 時間 |
| 距離估算  | 參照物比例尺                 | 手部軌跡 | 距離 (cm)       |
| SIMO 偵測 | 雙手追蹤                     | 手部軌跡 | 重疊時間        |

#### 邏輯軌道
| 元件       | 技術              | 輸入         | 輸出            |
|------------|-------------------|--------------|-----------------|
| SOP 解析器 | LLM + RAG         | SOP 文字     | 結構化步驟      |
| 參數預填   | Component DB 查詢 | Component ID | 預設 G/P/M/I 值 |
| 語法映射   | ddm_structure.csv | 動作動詞     | MOST 序列模板   |

### 5.3 階段三：融合計算

**優先級解析**:
```
優先級: 影片實測 > 元件資料庫 > SOP 文字推斷
```

| 參數     | 影片來源             | DB 來源               | 備援         |
|----------|----------------------|-----------------------|--------------|
| A (距離) | 手部軌跡 (cm → A 值) | typical_A             | SOP 推斷     |
| G (取得) | 物件尺寸偵測         | default_G             | G1           |
| P (放置) | 視覺難度             | default_P             | SOP 動詞映射 |
| M (移動) | 移動類型             | default_M             | SOP 動詞映射 |
| X (過程) | 碼錶計時             | machine_ref.cycle_sec | 固定值       |
| I (檢查) | 靜止 > 1 秒偵測      | default_I             | SOP 關鍵字   |

**TMU 計算**:
```
TMU = (A1 + B + G + A2 + B + P + A3) × 10 × Freq
其中: 1 TMU = 0.036 秒
```

### 5.4 階段四：驗證輸出

| 步驟 | 處理                 | 輸出                |
|------|----------------------|---------------------|
| 4.1  | 執行 VAL-01 ~ VAL-09 | 驗證報告            |
| 4.2  | 產生 Diff Report     | D-001 ~ D-XXX 項目  |
| 4.3  | 標記低信心度項目     | 審核佇列            |
| 4.4  | 匯出結果             | CSV, JSON, PDF 報告 |

| **第三階段：視覺核心**   | 第 5-6 週   | 手套手部偵測模型 (mAP > 0.8)、距離估算模組                           |
| **第四階段：邏輯與融合** | 第 7-9 週   | LLM Prompt 最佳化、多模態融合引擎                                    |
| **第五階段：驗收**       | 第 10-12 週 | 端到端測試、模型優化、部署                                            |
